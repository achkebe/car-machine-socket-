# 车机系统Socket通信模拟
Python实现的车机多模块跨进程通信模拟，基于TCP Socket实现总控、车速传感器、发动机、显示模块的分布式通信，贴合真实车载硬件的网络通信逻辑。

## 项目功能
1. 车速传感器：实时采集车速，响应发动机的车速请求，持续上报总控；
2. 发动机模块：主动向传感器请求车速，执行总控的启停/调转速指令，返回状态；
3. 显示模块：接收总控指令，终端模拟车机屏幕打印车速/发动机状态；
4. 总控模块：通信中枢，协调所有模块，模拟发动机控制流程。

## 技术栈
- Python 3.x
- TCP Socket 跨进程通信
- JSON 数据格式化传输
- 多线程处理并发连接/数据采集

## 运行步骤
### 1. 环境准备
无需额外依赖，Python 3.6+ 即可运行。

### 2. 启动顺序（必须按此顺序，避免连接失败）
1. 启动显示模块：`python display_module.py`
2. 启动发动机模块：`python engine_module.py`
3. 启动传感器模块：`python sensor_module.py`
4. 启动总控模块：`python main.py`

### 3. 运行效果
- 启动后每秒刷新车速并在车机显示；
- 3秒后发动机启动，显示状态+转速+车速；
- 8秒后发动机转速调至2500，更新显示；
- 12秒后发动机熄火，2秒后车机系统停止工作。
